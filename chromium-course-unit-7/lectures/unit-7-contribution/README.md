# Unit VII — Contribution & Ecosystem

**Week 14 · Lectures 27–28**

This unit closes the course with two questions:

- **L27**: How do you actually get code into Chromium — from finding a bug to landing a CL — and how do you engage with the broader professional ecosystem from zero to deep?
- **L28**: What is the Chromium ecosystem beyond the browser itself, and where is it going?

## Unit learning outcomes

Students who complete Unit VII can:

1. Find a Chromium bug suitable for a first contribution and describe the steps to fix it.
2. Navigate crbug.com, chromium-review.googlesource.com, and chromestatus.com.
3. Describe the Blink "Intent to Prototype / Ship" process and why it exists.
4. List at least five ways to contribute to the web platform ecosystem at different skill and time levels.
5. Explain the relationship between Chromium, Chrome, Edge, Brave, Electron, and CEF.
6. Describe current governance tensions in the web platform ecosystem.
